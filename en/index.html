jformValidate.js

Simple validation forms

Download ZIP

jformValidate.js v1.0

This plugin was developed considering existing complex libraries in the market , whenever we need to validate forms on the client-side , we face no plugins, but which in fact we choose?

Thinking that way and losing a lot of time implementing validation in forms, I decided to write my own plugin, behold, the jformValidate.js , comes with it is possible to place and remove validations of forms very simple , and you can also disable just changing a value , without the need to write javascript .

Using jformValidate.js library is also possible to add new types of custom validations, or if you prefer you can use those already implemented by default , download and implement in their application and rate.


Dependencies

The jformValidate.js plugin was developed based on the jQuery library , with the minimum requirement version 1.7 , if you do not have it in your project , visit the official site and download now on jQuery.com


Installation

To install the jformValidate.js library, download and add before closing the body of your html, if you want, the package is available at bower , or click the download link on top of the site. 

            
bower install jformvalidate

Implementation

            
<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="UTF-8">
		<title>Design Example</title>
	</head>

	<body>

        <!-- Body the code... -->

		<script src="{{PATH}}/jform.validate-1.0.0.js"></script>
        <script>
            // Initializing the library
            jformApp.init();
        </script>
	</body>
</html>

Configuration Parameters

            
jformApp.init({
    message: {
        text: {
            // Definition of messages to the verification method
            empty: 'Field is required!',
            cnpj: 'Enter a valid cnpj',
            cpf: 'Enter a valid cpf',
            email: 'Enter a valid email',
            url: 'Please enter a valid url', 
            cep: 'Enter a valid zipcode', 
            minlength: 'You need a minimum of letters', 
            number: 'Insert only decimal numbers', 
            extension: 'Invalid extension', 
            letters: 'Insert only letters'
        },
        style: {
            // Class definition for the error message element.
            this: {
                // Sets the class name
                name: 'jform-msg-error',
                // Specifies that the name is a class
                type: '.'
            },

            // Class definition for the element returned error
            element: {
                name: 'jform-elem-error',
                type: '.'   
            },

            // Class definition for the form that contains errors.
            form: {
                name: 'jform-form-error',
                type: '.' 
            }
        }
    }
});

After making the call, please use it there through date attributes inserted in your html tags.


Ways to use the meta tags

            
<!-- Only allowed letters -->
<input type="text" data-jform="letters" data-jform-message="Mensagem">

<!-- Only a valid zipcode format is allowed -->
<input type="text" data-jform="cep" data-jform-message="Mensagem">

<!-- Only numbers are allowed -->
<input type="text" data-jform="number" data-jform-message="Mensagem">

<!-- 
It is only allowed the extensions specified in the data attribute option,
separated by '|', for example: {{extensão}} | {{extensão}}
-->
<input type="file" data-jform="extension" data-jform-option="xls|csv" data-jform-message="Mensagem">

<!-- Only a valid URL format is allowed -->
<input type="text" data-jform="url" data-jform-message="Mensagem">

<!-- Field select mandatory, and the value of different empty -->
<select data-jform>
    <option value="">Select</option>
    <option value="01">test</option>
</select>

<!-- It is mandatory to select an option -->
<div data-jform="radio">
    <input type="radio" name="text" value="0">
    <input type="radio" name="text" value="1">
    <input type="radio" name="text" value="2">
    <input type="radio" name="text" value="3">
    <input type="radio" name="text" value="4">
</div>

<!-- 
When specifying the option field and a value is nescessário to be selected
at least 2 options, if not declared the date option, by default will be a
Required item
-->
<div data-jform="checkbox" data-jform-option="2">
    <input type="checkbox" name="vehicle" value="0">I have a bike
    <input type="checkbox" name="vehicle" value="1">I have a bike
    <input type="checkbox" name="vehicle" value="2">I have a bike
    <input type="checkbox" name="vehicle" value="3">I have a bike
    <input type="checkbox" name="vehicle" value="4">I have a bike
    <input type="checkbox" name="vehicle" value="5">I have a bike
</div>

<!-- Only a valid email format is allowed -->
<input type="text" data-jform="email" data-jform-message="Mensagem">

<!-- Only valid cnpj numeration is allowed -->
<input type="text" data-jform="cnpj" data-jform-message="Mensagem">

<!-- Only valid cpf numeration is allowed -->
<input type="text" data-jform="cpf" data-jform-message="Mensagem">

<!-- 
By setting the date option you will be setting the minimum character
to be entered 
-->
<input type="text" data-jform="minlength" data-jform-option="5" data-jform-message="Mensagem">

<!-- It can not be empty, mandatory field -->
<input type="text" data-jform>

<!-- 
At the end of your form, the submit button, you should tell submit date
to specify the plugin that this form will go through the validation.

In this attribute data, you have two options, true (Enable validations) or false (disables validations)
if you want to test the sending of the form without having to be putting certain values ​​to pass validations
-->
<button type="submit" data-jform-submit="true">Register</button>

Adding custom methods

To this add the following script tag that you call the library, the jform.additional.js 

            
<script src="{{PATH}}/jform.validate-1.0.0.js"></script>
<script src="{{PATH}}/jform.additional.js"></script>

Then, within the file jform.additional.js you will create its function, she will receive a parameter which will in their element, it is starting that will be validated, at the end of your code this function must return true to pass validation or false to return error. 

            
function myFunction(element){
    return false || true;
}

Now just call in your html tag, as follows: 

            
<!-- 
By choosing to create a custom validation, you must specify a date function method
which will have its validation without informing '( )', should only be placed the name
-->
<input type="text" data-jform="addmethod" data-jform-method="myFunction" data-jform-message="Mensagem">

Available methods

If you want to use pre-defined methods in its custom validations, simply call in your function method. 

            
// The field can not be empty
jformApp.validate.empty(elemento);

// The value entered must be a valid cnpj
jformApp.validate.cnpj(elemento);

// The value entered must be a valid cpf
jformApp.validate.cpf(elemento);

// The value entered must be a valid email address
jformApp.validate.email(elemento);

// The value entered must be a valid email address
jformApp.validate.url(elemento);

// The value entered must be a valid zipcode
jformApp.validate.cep(elemento);

// The value entered must be only numbers
jformApp.validate.number(elemento);

// The value should be informed only letters
jformApp.validate.letters(elemento);
