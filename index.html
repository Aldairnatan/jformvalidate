jformValidate

Simples validação de formulários

Download ZIP

jformValidate v1.0

Este módulo foi desenvolvido pensando em plugins complexos existentes em mercado, sempre quando precisamos validar formulários no client-side, nos deparamos com n plugins, mas qual de fato vamos escolher?

Pensando dessa forma e perdendo muito tempo implementando validações em formulários, que decidi escrever a minha própria biblioteca o jformValidate, com ele é possível colocar e remover validações do formulário de forma muito fácil, sendo possível também desabilitar apenas alterando um valor, sem ter a necessidade de escrever javascript.

Usando a biblioteca jformValidate é possível também incluir novos tipos de validações personalizadas, ou se preferir usando as que já existem implementadas por padrão, baixe e implemente na sua aplicação e avalie.


Dependências

O módulo jformValidate foi desenvolvido baseado na biblioteca jQuery, tendo como requisito minímo a versão 1.7, caso não possua-o em seu projeto, visite o site oficial e baixe agora em jQuery.com


Instalação

Para instalar a biblioteca jformValidate, baixe e inclua antes de fechar o body do seu html, caso queira, o pacote está disponível no bower, ou clique no link de download no topo do site. 

            
bower install jformvalidate

Implementação

            
<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="UTF-8">
		<title>Estrutura Exemplo</title>
	</head>

	<body>

        <!-- Corpo de código... -->

		<script src="{{PATH}}/jform.validate-1.0.0.js"></script>
        <script>
            // Inicializando a biblioteca
            jformApp.init();
        </script>
	</body>
</html>

Parâmetros de configuração

            
jformApp.init({
    message: {
        text: {
            // Definição de mensagens para os método de verificação
            empty: 'Este campo é obrigatório!',
            cnpj: 'Insira um cnpj válido',
            cpf: 'Insira um cnpj válido',
            email: 'Insira um email válido',
            url: 'Insira uma url válida', 
            cep: 'Insira um cep válido', 
            minlength: 'Você deve por um mínimo de letras', 
            number: 'Insira somente números decimais', 
            extension: 'Extensão inválida', 
            letters: 'Insira somente letras'
        },
        style: {
            // Definição de classe para o elemento mensagem de erro.
            this: {
                // Define o nome da classe
                name: 'jform-msg-error',
                // Especifica que o name será uma classe
                type: '.'
            },

            // Definição de classe para o elemento que retornou erro
            element: {
                name: 'jform-elem-error',
                type: '.'   
            },

            // Definição de classe para o formulário que contém erros.
            form: {
                name: 'jform-form-error',
                type: '.' 
            }
        }
    }
});

Após fazer a chamada, basta usa-lá através de data atributos inseridos nas suas tags html


Formas de uso das meta tags

            
<!-- Somente é permitido letras -->
<input type="text" data-jform="letters" data-jform-message="Mensagem">

<!-- Somente é permitido um formato de cep válido -->
<input type="text" data-jform="cep" data-jform-message="Mensagem">

<!-- Somente é permitido números -->
<input type="text" data-jform="number" data-jform-message="Mensagem">

<!-- 
Somente é permitido as extensões especificadas no data atributo option, 
separando por '|', exemplo: {{extensão}} | {{extensão}}
-->
<input type="file" data-jform="extension" data-jform-option="xls|csv" data-jform-message="Mensagem">

<!-- Somente é permitido um formato de url válido -->
<input type="text" data-jform="url" data-jform-message="Mensagem">

<!-- Campo select obrigatório, e o value diferente de vazio -->
<select data-jform>
    <option value="">Selecione</option>
    <option value="01">teste</option>
</select>

<!-- É obrigatório selecionar uma opção -->
<div data-jform="radio">
    <input type="radio" name="text" value="0">
    <input type="radio" name="text" value="1">
    <input type="radio" name="text" value="2">
    <input type="radio" name="text" value="3">
    <input type="radio" name="text" value="4">
</div>

<!-- 
Ao especificar o campo option e um valor, é nescessário que seja selecionado 
no mínimo 2 opções, se não for declarado o data option, por default será um 
item obrigatório
-->
<div data-jform="checkbox" data-jform-option="2">
    <input type="checkbox" name="vehicle" value="0">I have a bike
    <input type="checkbox" name="vehicle" value="1">I have a bike
    <input type="checkbox" name="vehicle" value="2">I have a bike
    <input type="checkbox" name="vehicle" value="3">I have a bike
    <input type="checkbox" name="vehicle" value="4">I have a bike
    <input type="checkbox" name="vehicle" value="5">I have a bike
</div>

<!-- Somente é permitido um formato de email válido -->
<input type="text" data-jform="email" data-jform-message="Mensagem">

<!-- Somente é permitido númeração de cnpj válido -->
<input type="text" data-jform="cnpj" data-jform-message="Mensagem">

<!-- Somente é permitido númeração de cpf válido -->
<input type="text" data-jform="cpf" data-jform-message="Mensagem">

<!-- 
Ao definir o data option você estará definindo o mínimo de caracteres 
que deverá ser digitado 
-->
<input type="text" data-jform="minlength" data-jform-option="5" data-jform-message="Mensagem">

<!-- Não poderá ser vazio, campo obrigatório -->
<input type="text" data-jform>

<!-- 
No final do seu formulário, no botão de submit, você deverá informar o data submit,
para especificar ao módulo que este formulário irá passar pelas validações.

Nesse data atributo, você terá duas opções, true(Habilitar as validações) ou false(Desabilita as validações),
caso queira testar o envio do formulário sem ter que ficar colocando valores certos para passar as validações
-->
<button type="submit" data-jform-submit="true">Cadastrar</button>

Adicionando métodos customizados

Para isso adicione abaixo da tag script que você chamou a biblioteca, o arquivo jform.additional.js 

            
<script src="{{PATH}}/jform.validate-1.0.0.js"></script>
<script src="{{PATH}}/jform.additional.js"></script>

Feito isso, dentro do arquivo jform.additional.js você irá criar a sua função, ela irá receber um parâmetro no qual terá o seu elemento, que é apartir dele que irá ter a validação, ao finalizar o seu código essa função devera retornar true para se passar na validação ou false se retornar erro. 

            
function minhaFuncao(element){
    return false || true;
}

Agora basta chamar na sua tag html, da seguinte forma: 

            
<!-- 
Ao escolher criar uma validação personalizada, você deverá especificar em data method a função
que irá ter a sua validação, sem informar '( )', somente deverá ser colocado o nome
-->
<input type="text" data-jform="addmethod" data-jform-method="minhaFuncao" data-jform-message="Mensagem">

Métodos disponíveis

Caso queira usar métodos pré definidos nas suas validações personalizadas, basta chamar dentro da sua função o método. 

            
// O campo não poderá ser vazio
jformApp.validate.empty(elemento);

// O valor informado deverá ser um cnpj válido
jformApp.validate.cnpj(elemento);

// O valor informado deverá ser um cpf válido
jformApp.validate.cpf(elemento);

// O valor informado deverá ser um email válido
jformApp.validate.email(elemento);

// O valor informado deverá ser um email válido
jformApp.validate.url(elemento);

// O valor informado deverá ser um cep válido
jformApp.validate.cep(elemento);

// O valor informado deverá ser somente números
jformApp.validate.number(elemento);

// O valor informado deverá ser somente letras
jformApp.validate.letters(elemento);
